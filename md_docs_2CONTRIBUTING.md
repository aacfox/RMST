<ul>

</ul>
</li>

<p>As one can see, almost everything is done. Each checked subroutine is O(n), afak. <br  />
 Every subroutine, imho, deserves its own section with eleborate description of how and why it works. Probably in the form of a wiki, or just here in place. <br  />

<h1><a class="anchor" id="autotoc_md1"></a>
Test cases</h1>
<p>Every test case are made of two files: one with <code>.csv</code> extension, one with <code>.mm</code>. <br  />
 <code>.csv</code> files <a href="../README.md#limitations-so-far">was already explained.</a>. As for <code>.mm</code>, it stands for <code>mermaid</code> — diagramm creating language, including graphs. <br  />
 Don't worry! Nothing for you to learn. <code><a class="el" href="classGraph.md">Graph</a></code> already has a member function <code>mermaid()</code> dedicated to producing such files. <br  />
 Their contained code properly pasted in <code>.md</code> files will render a pretty graph! One of very few mean to visualize graphs.</p>
