<div class="textblock"><p><a class="anchor" id="md_README"></a>C++ Implementation of <a href="./README.md#randomized-om-mst-algo">a randomized expected linear time MST algorithm by D. Karger et al(1995)</a>. <br  />
 </p><blockquote class="doxtable">
<p>&zwj;[!caution] <br  />
 Time complexity is currently semilinear because I'm struggling with the last of the subroutines of the algorithm. <br  />
 </p>
</blockquote>
<p>For more eleborate discussion you may directly follow to <a href="https://aacfox.github.io/RMST/">the wiki</a>. <br  />
</p>
<h1><a class="anchor" id="autotoc_md5"></a>
Introduction</h1>
<p>For I am Constantine, the 'C' <strike>in C++</strike> in my nickname! And this is my half-assed but whole-hearted attempt to understand and implement the mentioned algo. I can hardly explain why of all the probable topics I deem this the most exciting, so here we go. <br  />
 In the wild, there exist other deadly cool algos for the same purpose (which is finding MSTs[^1] of graphs, in case you forgot) but this one just rubs me the right way and was easier to comprehend and finally implement. <a href="./README.md#other-cool-mst-algos">Others</a> are deterministic with time complexities being great always without degenerating in worst cases. <br  />
 [^1]: <a href="https://en.wikipedia.org/wiki/Minimum_spanning_tree">Minimum spanning tree</a> (or minimum spanning forest - which is a more general notion). <br  />
</p>
<h1><a class="anchor" id="autotoc_md6"></a>
What it does</h1>
<p>Eats graph, poos its MSF[^1] in the same format! <br  />
</p>
<h2><a class="anchor" id="autotoc_md7"></a>
Limitations so far</h2>
<p><a href="./mst_generator.exe">The program</a> supports only input files with <code>.csv</code> extension and formatted as <em><b>edge list</b></em> with <em><b>semicolon</b></em> as a delimiter and <em><b>integral</b></em> weights. See <a href="./docs/wikipedia_graph.csv">example</a>. To fulfill more complicated tasks source code provides more powerful tools not included in the executable ~~'cause of my lazy ass~~ to keep things simple and focus resources of humankind on <a href="./docs/CONTRIBUTING.md#tasks">much bigger problem</a>. <br  />
</p>

Get started</h1>
<p>The simplest case:</p><ol type="1">

</li>
<li>Run program.<ul>

<li>Otherwise press <code>1</code>, <code>Enter</code>, paste the path from the clipboard, <code>Enter</code>. <br  />
 If the message "Success!" popped, press <code>3</code>, then <code>Enter</code>.</li>
</ul>
</li>
<li>If you want to proccess all found files at once:<ul>

<li>proceed to step 4.</li>

</li>
<li>Otherwise to process one by one:<ul>

<h2><a class="anchor" id="autotoc_md9"></a>
The algorithm's source code</h2>
<p>The actual implementation of the algorithm is in a <a href="./graph.cpp">separate file</a>, designated as a member function template <code>mst()</code> of a class template named <code><a class="el" href="classGraph.md">Graph</a></code>, parameterized by the maximum theoretically possible <code>number of vertices</code> in a given graph for a given application, and by <code>type</code> representing weights of edges, respectively.</p>
<p>Any given graph is stored in a data member as a plain <code>vector&lt;<a class="el" href="structGraph_1_1Edge.md">Graph::Edge</a>&gt;</code> representing an edge list. But <code>mst()</code> is oblivious of this fact, so feel free to change the storage until a type-complete instance of <code><a class="el" href="classGraph.md">Graph</a></code> constitutes the <code>std::ranges::range</code> concept, and the derefencing of iterators returns <code><a class="el" href="structGraph_1_1Edge.md">Graph::Edge</a></code>. <br  />
</p>
<p><code><a class="el" href="classGraph.md">Graph</a></code> also has several <code>named constructors</code>. Specifically:</p><ul>
<li><code><a class="el" href="classGraph.md#ab7ca595c41bf323d1a71b705f82442c7" title="Named ctor">Graph::from_grid</a></code> — creates a graph from any 2D-range (i.e. supporting two consecutive subscript <code>operator[]</code>)</li>
<li><code><a class="el" href="classGraph.md#a79f5662987d8c62edd32365406bffcaa" title="Named ctor">Graph::from_csv</a></code> — creates a graph from any <code>std::istream</code> formatted as explained above</li>
</ul>
<p>Check <a href="./graph.cpp">source code</a> for the full signature and documentation. <br  />
</p>
